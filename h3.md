## c) Crack. Crack the hash 6b1628b016dff46e6fa35684be6acc96 .

Following Tero's guide for password cracking with Hashcat, I started by updating my package lists with: 

``sudo apt-get update``

After which I installed the necessary packages, hashcat, hashid & wget with the following command:

``sudo apt-get -y install hashid hashcat wget``

I then created a folder where to download the dictionary file

``mkdir h3``

and navigated inside with

``cd h3``

I then proceeded to download the dictionary that we will use, in this case rockyou

``wget https://github.com/danielmiessler/SecLists/raw/master/Passwords/Leaked-Databases/rockyou.txt.tar.gz``

Extracting the file with

``tar xf rockyou.txt.tar.gz``

and removing the redundant tar-file after extraction

``rm rockyou.txt.tar.gz``

After this, I used hashcat to figure out the hashing algorithm. 

``hashid -m 6b1628b016dff46e6fa35684be6acc96``

Which returned a list of different algorithms. Based on the top three results MD2, MD5 and MD4, I decided to test out MD5 (Hashcat Mode = 0) first, as suggested by Tero in his guide (much more common than MD2 or MD4).

``hashcat -m 0 '6b1628b016dff46e6fa35684be6acc96' rockyou.txt -o solved``

This is where I ran into an issue. The terminal returned an error **"Device #1: Not enough allocatable device memory for this attack"**. So it seems I have a RAM limitation. I temporarily shut down my VM to increase the RAM from 2 GB to 4 GB to see if this would resolve the issue. After rebooting the VM, I ran the command again and this time it was executed succesfully. Taking a look at the output file "solved" with

``cat solved``

the password was revealed to be

``6b1628b016dff46e6fa35684be6acc96:summer``

...So the password in this case was "summer"!
